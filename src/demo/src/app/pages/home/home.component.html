ngx-svg

Create powerful SVG objects with Angular. Based on svg.js library.

Getting Started

ngx-svg contains all core svg.js components, so you won't need to include any other external dependencies.

Installation

To install ngx-svg, you have to run the following command from the terminal (you have to be within the project folder) - npm install ngx-svg --save

This will install the latest version of ngx-svg and add it to your project dependency list.

After the installation you have to include the component in your module by importing it using the following command - import {{ '{' }} NgxSvgModule {{ '}' }} from 'ngx-svg';

Finally you have to add the NgxSvgModule to the imports of your module:

@NgModule({{ '{' }}
 declarations: [
  ...
 ],
 imports: [
  ...
  NgxSvgModule
 ],
 providers: [..],
 bootstrap: [...]
{{ '}' }})
export class AppModule {{ '{' }} {{ '}' }}

Afterwards you should be able to use the library in your application.

Container

To start using the ngx-svg you must declare a container object in one of your views, by using the following element:

<svg-container containerId="svg-element">
 ...
</svg-container>

In the below table you can see explanation of the input and output parameters.

ID Name Type Explanation Mandatory Default value
1 containerId Input parameter (string) Unique ID for the svg container. Will be used for drawing the svg elements. YES undefined
2 height Input parameter (number) Height of the container in pixels. NO 200
3 hoverable Input parameter (boolean) Indicator if user should be able to see dot on hover, to capture coordinates. NO false
4 showGrid Input parameter (boolean) Indicator if grid should be shown in the background of svg container. NO false
5 grid Input parameter (width: number, height: number, color: string) Grid object that will be used to configure grid. NO { width: 10, height: 10, color: #000 }
6 pointSize Input parameter (number) Numeric value in pixels, to indicate how large should the point be. NO 10
7 viewBox Input parameter (array of numbers) Viewbox of the container, must be an array consisting of 4 integers [x, y, width, height]. NO []
8 clickEvent Output parameter (x: number, y: number) Is fired when click event happens on the dot element. Works only if hoverable input parameter is enabled. NO N/A
9 doubleClickEvent Output parameter (x: number, y: number) Is fired when double click event happens on the dot element. Works only if hoverable input parameter is enabled. NO N/A
10 mouseOverEvent Output parameter (MouseEvent) Is fired when mouse is moved over the container. NO N/A
11 mouseOutEvent Output parameter (MouseEvent) Is fired when mouse exits the container area. NO N/A
12 mouseMoveEvent Output parameter (x: number, y: number) Is fired when mouse moves within the container area. NO N/A

Within one container you can enter as many elements as you want.

Custom CSS Classes

If you are using custom css classes, make sure that you define them using ::ng-deep combinator, otherwise ngx-svg components won't see the classes.

To avoid leaking styles to other components, make sure to use :host combinator as well.

You can find more information on the official Angular documentation.
